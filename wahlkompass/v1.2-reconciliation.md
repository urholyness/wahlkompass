# v1.2 Code Reconciliation — 2026-07-16

The code baseline was written against design **v1.1**; the consolidated design
was then revised to **v1.2** *after* the code (file timestamps confirm it). This
pass brings the implemented code up to v1.2 and adds the first pieces of the
scoring engine + integrity suite. All 28 pipeline tests green (1 skip = release
reports that need real evidence data).

## Fixed (verified bugs from the v1.1 baseline)

1. **Roll-call direction recode** — `pipeline/src/ingestors/bundestag_xml_parser.py`
   - direction is now `(Ja − Nein) / (Ja + Nein)` — *active votes only* (was
     `(Ja − Nein) / (Ja + Nein + Enthaltung)`, which diluted direction toward 0
     — the exact Enthaltung-as-0 behavior v1.2 §3.2 repudiated).
   - emits participation-share `item_weight = (Ja + Nein) / (Ja + Nein + Enthaltung)`.
   - **unified abstention**: participation `< 0.10` ⇒ item excluded
     (`item_weight = 0`, `excluded_unified_abstention = true`), logged for the
     evidence drawer as *geschlossene Enthaltung*.
   - FCI unchanged (`max(Y,N,E)/(Y+N+E)`), still display-only.

2. **`item_weight` applied in aggregation** — `pipeline/src/aggregation.py`
   - effective weight is now `w = τ · ρ · item_weight` in the numerator,
     denominator, and confidence variance (was: `item_weight` only used to
     *exclude* an item when exactly 0, so a 0.5 share silently counted as 1.0).
   - excluded items (w = 0) also no longer count toward the admission rule.

3. **Independent verifier hardened** — `tools/reproduce.py`
   - same `item_weight` fix (kept as a *separate* re-implementation on purpose).
   - reads methodology constants + target date from the release
     (`methodology.json` or `meta.methodology`, `meta.as_of`); warns + falls back
     to deu-bundestag-21 defaults only when the release fails to pin them.
   - **exact (byte-identical) comparison** on p/confidence/p_said/p_did/divergent
     (was a 0.001/0.01 tolerance that hid a wrong stored confidence).

## Added (progress beyond the reconciliation)

4. **Reference scoring engine** — `pipeline/src/scoring.py` (new)
   - `match_score` = normalized weighted-L1 `S(P)`; display half-width `h(P)`
     (RSS, z=1) and conservative tie bound `h_max(P)` per v1.2 §2.3.
   - `rank_parties` = score-ordered ranking with **ties rendered as ties**
     (shared rank when `|ΔS| < max(h_max)`), low-coverage flag (<10 answers).
   - This is the Python reference the TypeScript `engine/` will be cross-checked
     against byte-for-byte in CI (§14.3).

5. **§9 bias & integrity suite** — `pipeline/tests/test_bias_suite.py` (new)
   - determinism/golden, **party-symmetry** (relabel ⇒ ranks permute identically),
     **mirror** (u vs −u on mirrored positions ⇒ identical S), **no-imputation**
     (null position removed from num *and* denom; source-scan for default
     branches). Plus `test_scoring.py` unit tests.
   - Release-report checks (coverage-parity §9.5, statement-balance §9.6,
     divergence-symmetry §9.8) are stubbed/skipped — they need real evidence data
     and wire in at the first release build.
